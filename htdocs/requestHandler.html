Request Handler

The RequestHandler is responsible for handling a single HTTP GET request per instance. The Shuttle daemon will create an instance of the RequestHandler and pass the request in. The RequestHandler should parse the request, along with any parameters, and then return the requested page from either the cache or the file system. If the page is not found, or another error occurs, the RequestHandler is responsible for conveying the issue with standard HTTP codes.
